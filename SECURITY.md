# Security Policy

## Reporting a vulnerability
Please use GitHub's private vulnerability reporting for this repository:
- Security tab -> Advisories -> Report a vulnerability

Do not open public issues for suspected vulnerabilities.

## What to include
- Affected file(s) and component(s)
- Reproduction steps
- Impact assessment
- Suggested remediation (if known)

## Response targets
- Initial triage acknowledgement: within 5 business days
- Remediation plan or status update: within 10 business days

## Scope
In scope:
- Credential leaks
- Privilege escalation paths
- Remote code execution vectors
- Network exposure misconfigurations
- Supply-chain/script trust issues

Out of scope:
- Social engineering
- Vulnerabilities in third-party services not controlled by this repo
